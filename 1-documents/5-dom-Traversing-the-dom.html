DOM: Traversing the DOM

To get access to an Element(object) the document gives us a few options

going Up

node.parentNode node.parentElement

These are identical but the parentElement

only returns a node if the parentElement is an element

document.body.parentNode; // the html element
document.body.parentElement; // the html element

document.documentElement.parentNode; // the document node
document.documentElement.parentElement; // null

(document.documentElement.parentNode === document);  // true
(document.documentElement.parentElement === document);  // false

The property can be chained together to keep moving up

nodeElement.parentNode.parentNode.parentNode

If you kept going you will eventually hit the body element.

If you go to far you will get null.

going Down

node.childNodes node.children

These are identical but the children only return elements in an array.

Where as childNodes return all nodes in an array

childNodes will include text nodes and other non element nodes as well.

The property can be chained together to keep moving down

nodeElement.children[0].children[0].children[0]

will keep returning the first element inside of each element

If you go to far you will get null.
